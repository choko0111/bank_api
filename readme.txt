Что вообще происходит: у нас есть модели запросов (request) и модели ответов (response), и нам нужно сравнивать их поля после API-вызова.

Request объект
     │
     ▼
ComparisonRule
Хранит правила соответствия между полями Request и Response.
Словарь вида: {request_field: response_field}.
Нужно, чтобы знать какие поля сравнивать и с чем.
     │
     ▼
ModelComparator
Берет Request и Response и сравнивает поля по правилам из ComparisonRule.
Создает список несоответствий (Mismatch) для полей, которые не совпали.
Это логика сравнения моделей.
     │
     ▼
ComparisonResult (список mismatches)
Объект-результат работы Comparator.
Хранит список всех несовпадений (mismatches).
Метод is_success() позволяет быстро проверить: тест прошел или нет.
     │
     ▼
ModelAssertions.raise_if_failed()
Удобная обертка для тестов.
Вызывает Comparator, получает результат и кидает AssertionError, если есть несоответствия.
